2 Latar Belakang

Pemahaman Project

Analisis kebutuhan dan tujuan dari project website Sekolahku.

Sekolahku (SekolahNews) adalah platform portal berita berbasis digital yang akan menggantikan fungsi majalah dinding (mading) konvensional sekolah menjadi media online terpadu yang interaktif.

Di era digital saat ini, kehadiran online menjadi kebutuhan mutlak bagi setiap bisnis. Konsumen modern menginginkan kemudahan berbelanja dari rumah dengan jaminan kualitas produk yang terpercaya. Website e-commerce menjadi solusi tepat untuk menjembatani penjual dan pembeli secara efisien, sekaligus memberikan kontrol penuh kepada pemilik bisnis melalui sistem CMS (Content Management System) untuk mengelola produk, konten, dan transaksi secara mandiri.

Melihat potensi besar ini, kami dari BuildyWeb × DigiNetAll siap mewujudkan visi Sekolahku menjadi platform portal sekolah yang profesional, menarik, dan mudah digunakan — dibangun dengan teknologi Laravel & Inertia.js yang terbukti handal, sehingga mampu meningkatkan kepercayaan calon pelanggan dan mendorong penjualan.

lightbulb Nilai Tambah: Desain UI / UX profesional dan formal yang kami bangun sangat merepresentasikan nilai institusi pendidikan dan mudah digunakan oleh seluruh warga sekolah.

3 Ruang Lingkup

Cakupan Pekerjaan

Detail layanan dan fitur yang akan kami kerjakan dalam project ini.

B. Pilihan Stack Teknologi

Paket 1: Laravel
Pengembangan dengan full-stack Laravel Native. Tangguh, aman, dan cocok untuk deployment mandiri di hosting konvensional. (Khusus Source Code)
Paket 2: Next.js
Pengembangan dengan framework React / Next.js terkini. Super cepat dengan performa SEO terbaik. (Sudah termasuk Hosting & Domain)

C. Fitur Teknis

  • check_circle
    CMS (Content Management System) — Panel admin untuk mengelola produk, kategori, konten, dan data secara mandiri tanpa harus coding
  • check_circle
    SPA Experience — Navigasi cepat tanpa reload halaman menggunakan Inertia.js, terasa seperti aplikasi native
  • check_circle
    Fully Responsive — Tampil sempurna di semua ukuran layar (mobile, tablet, desktop)
  • check_circle
    Secure & Scalable — Dibangun di atas Laravel yang memiliki fitur keamanan bawaan (CSRF, SQL injection protection, dll) dan mudah di-scale
  • check_circle
    SEO-Friendly — Server-side rendering untuk indexing optimal di mesin pencari

6 Timeline

Jadwal Pengerjaan

Project akan diselesaikan dalam 3 hari kerja sejak pembayaran DP diterima dan brief disetujui.

1
Fase Persiapan
Setup Environment & Sinkronisasi Source Code
Konfigurasi awal database, pemilihan varian tech stack (Laravel/Next.js) dan instalasi struktur dasar prototipe portal sekolah.
2
Fase Pengembangan Inti
Implementasi CMS dan Fitur Konten
Pemasangan fungsi berita (CRUD Editor), sistem Galeri Masonry, dan integrasi struktur Kesiswaan (OSIS). (Maksimal Hari ke-2).
3
Fase Final (Maks. Hari ke-3)
Pengujian, Hosting & Penyerahan
Apabila memilih paket komplit, sistem dipindahkan live ke Hosting, konfigurasi server name Domain, checking error & loading speed, serta serah terima kepada pihak manajemen sekolah.
7 Ketentuan

Syarat & Ketentuan

Ketentuan yang berlaku selama dan setelah pengerjaan project.

  1. Proposal ini berlaku selama 30 hari sejak tanggal dikeluarkan (hingga 26 April 2026). Setelah masa berlaku habis, harga dan ketentuan dapat berubah.
  2. Pembayaran dilakukan dalam 2 tahap: 50% sebagai DP sebelum pengerjaan dimulai, dan 50% setelah project selesai dan disetujui.
  3. Pembeli Paket Komplit (Next.js) akan mendapatkan hak pemeliharaan server ringan dan pembaruan lisensi domain gratis selama 1 tahun pertama.
  4. Klien akan mendapatkan prioritas bantuan konsultasi bug apabila portal mengalami error saat digunakan setelah serah terima tanpa biaya tambahan (Garansi Live Berjalan 30 hari).
  5. Proses pengerjaan sistem aplikasi memiliki tenggat waktu (SLA) maksimal 3 hari kerja terhitung saat DP diterima.
  6. Deliverables berupa hak penuh akses admin login. Akses file codebase (Source Code) dikirimkan menggunakan Github / ZIP sesuai preferensi pihak yayasan sekolah.
  7. Seluruh data konten bawaan yang kami upload bersifat uji coba (Dummy data). Data valid akan dikelola langsung oleh administrator sekolah.
  8. BuildyWeb × DigiNetAll berhak menampilkan project ini sebagai portfolio, kecuali klien merasa keberatan, silakan sampaikan di awal.
